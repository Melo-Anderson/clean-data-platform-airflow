# YAML Data Engineering Repo Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create Phase 2 of our architecture — a completely separate Data Engineering repository (`airflow-data-yamls`) that holds pipeline configurations and uses Option B (Docker) to validate them locally.

**Architecture:** A new folder next to our core platform. It will have a `Makefile` that pulls the core platform Docker image (or builds it locally for dev) to run the CLI.

**Tech Stack:** Make, Docker, YAML.

## Global Constraints

- The new repository must be created one level up, parallel to the core platform (e.g., `../airflow-data-yamls`).
- The `Makefile` must use Docker to run the validation (Option B).
- Do not write Python code here; only YAML configuration and Makefiles.
- **Localhost only**: All example connections and dataset URIs must point to localhost or local files.

---

### Task 1: Initialize the Data Engineering Repository

**Files:**
- Create: `../airflow-data-yamls/Makefile`
- Create: `../airflow-data-yamls/yamls/example_pipeline.yaml`
- Create: `../airflow-data-yamls/README.md`

**Interfaces:**
- Consumes: The Docker image `platform-api:latest` (built by the core repo).
- Produces: A ready-to-use Data Engineering repository.

- [ ] **Step 1: Create the directory structure**

```bash
mkdir -p ../airflow-data-yamls/yamls
```

- [ ] **Step 2: Write the Makefile**

Create `../airflow-data-yamls/Makefile`:
```makefile
.PHONY: validate validate-dry-run

CLI_IMAGE ?= platform-api:latest

# We mount the local yamls/ folder into the container
validate:
	docker run --rm -v $(PWD)/yamls:/app/dags/yamls $(CLI_IMAGE) uv run cli pipeline rebuild dags/

validate-dry-run:
	docker run --rm -v $(PWD)/yamls:/app/dags/yamls $(CLI_IMAGE) uv run cli pipeline rebuild dags/ --dry-run
```

- [ ] **Step 3: Create a local-only example pipeline YAML**

Create `../airflow-data-yamls/yamls/example_pipeline.yaml`:
```yaml
pipeline:
  name: "example_ingestion_local"
  description: "A simple local ingestion pipeline"
  schedule: "0 0 * * *"
  type: "ingestion"
  dataset_uri: "postgres://localhost:5432/platform_db/public.users_landing"
  source:
    type: "database"
    connection_id: "postgres_default"
    query: "SELECT * FROM public.users"
  destination:
    type: "database"
    connection_id: "postgres_default"
    table: "public.users_landing"
```

- [ ] **Step 4: Create a simple README**

Create `../airflow-data-yamls/README.md`:
```markdown
# Data Engineering Pipelines (Local)

This repository stores all the YAML definitions for our data pipelines.

## How to validate locally

You must have Docker installed and the core platform API image built (`platform-api:latest`).

make validate-dry-run
```
```
